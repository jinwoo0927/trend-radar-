# 📡 TrendRadar

유튜브·틱톡·인스타그램·스레드의 인기 영상(조회수·좋아요·댓글·공유)을 수집해 **분야별로 자동 분류**하고 **트렌드를 분석**하는 마케팅 대시보드. 기본적으로 **100% 실데이터**만 다룹니다(가짜/데모 없음).

## 실행 방법

```bash
cd trend-radar
npm install
npm start
```

브라우저에서 http://localhost:3600 접속.

## 주요 기능

| 기능 | 설명 |
|------|------|
| 수집 | 서버 시작 시 자동 수집 + 60분 간격 자동 수집(다음 수집 카운트다운 표시) + "지금 수집" 버튼 + **수집 이력 테이블**(일시·건수·실데이터 수·소스·소요 시간, 최근 30회 기록) |
| 분류 | 제목·해시태그 키워드 매칭으로 12개 분야 자동 분류 (뷰티, 패션, 푸드, 테크·IT, 게임, 음악·댄스, 운동·건강, 여행, 교육·지식, 엔터테인먼트, 브이로그·라이프, 비즈니스·마케팅) |
| 분석 | 인게이지먼트율 = (좋아요+댓글+공유)/조회수, 급상승 점수(조회수 성장률 기준), 트렌드 키워드 추출, 이전 수집 대비 증감률, 자동 텍스트 인사이트(7종) |
| 날짜 | 모든 영상에 **게시일**(상대 표기 "5일 전"을 절대 날짜로 변환)과 **수집일시** 표시 — 카드·랭킹 테이블·수집 이력·차트 축 전부 |
| 필터·검색 | 플랫폼 탭, 분야 선택, 7가지 정렬(조회수/좋아요/댓글/공유/인게이지먼트/급상승/최신 게시순), **제목·채널·해시태그 검색**, 표시 개수(20/50/100), **키워드 클릭 필터** |
| 차트 | 분야별 조회수, 플랫폼 점유율, **분야별 평균 인게이지먼트율**, **수집일별 전체 조회수 추이**, 급상승 TOP5 성장 곡선 |
| 내보내기 | **CSV 다운로드** — 현재 필터·정렬 그대로, 엑셀 한글 호환(BOM), 게시일·수집일시·URL·데이터 출처 포함 |
| 미리보기 | 썸네일 카드 + 카드/랭킹 행 클릭 시 사이트 내 공식 임베드 재생 |
| **💡 콘텐츠 아이디어** | 공유·좋아요가 잘 되는 실영상에서 **잘 먹히는 형식·주제·해시태그를 역산**해 구체적 아이디어 카드로 제시. 모든 아이디어에 **근거 실영상**(지표 포함) 동반. 목표(좋아요형/공유형) 토글 + 분야·키워드로 좁히기 |
| **🔬 콘텐츠 해부 (AI)** | 랭킹의 각 영상/글에 "🔬 해부" 버튼 → Claude가 **훅·구조·바이럴 요인·겨냥 감정·재현 공식·적용 팁**을 실텍스트·지표 근거로 분석 |
| **✍️ 콘텐츠 스튜디오 (AI)** | ① 내 주제를 적으면 인기 콘텐츠 구조를 흡수해 **플랫폼별 대본(유튜브·틱톡)·글(스레드·인스타)** 완성 ② 내 초안을 붙여넣으면 **바이럴 스타일로 변환**(무엇을 왜 바꿨는지 설명). 참고 인기작 자동 선택, 결과 복사 |
| **📝 유튜브 대본 수집** | `yt-dlp`로 인기 영상의 **실제 자막(발화 대본)을 API 키 없이 무료** 추출 → 해부·스튜디오가 제목·설명이 아닌 **진짜 대본**을 분석·참고 (`pip install yt-dlp` 필요) |
| **💸 무료 AI 옵션** | AI 기능은 **무료 제공자로 비용 0** 운영 가능 — Google Gemini(무료 티어)·Groq(무료)·로컬 Ollama($0). `AI_PROVIDER`/키만 설정하면 자동 사용, 유료 Claude도 선택 가능 |

## 미리보기 · URL 등록

- **미리보기 카드** — 실데이터 영상은 썸네일 카드로 표시되고, 카드를 누르면 **사이트 안에서 공식 임베드 플레이어로 바로 재생**됩니다 (유튜브 `youtube-nocookie` / 틱톡 `embed/v2` / 인스타그램 `/embed/` / 스레드 `/embed`).
- **URL 등록** — 상단 입력창에 유튜브·틱톡·인스타그램·스레드 영상 URL을 붙여넣으면 **API 키 없이** 메타데이터(제목·작성자·썸네일, 유튜브는 조회수·좋아요까지)를 가져와 미리보기·랭킹·분석에 즉시 반영합니다. 틱톡 단축 링크(vt.tiktok.com)도 자동 해석됩니다.
- 썸네일은 `/api/thumb` 프록시(유튜브·틱톡 CDN 화이트리스트)를 거쳐 어떤 네트워크 환경에서도 안정적으로 표시됩니다.

## 데이터 소스 (API 키 불필요)

| 플랫폼 | 자동 수집 | 방식 |
|--------|----------|------|
| 유튜브 | ✅ 실데이터 | ① `YOUTUBE_API_KEY` 설정 시 공식 Data API → ② 키가 없으면 **공개 검색(이번 주 업로드 + 조회수순)을 분야별 12개 시드 쿼리로 파싱** + 시청 페이지에서 좋아요 보강. 2025년 유튜브 트렌딩 페이지 폐지 이후의 정공법. 키 없는 모드에서는 댓글 수 미제공(`–` 표시) |
| 틱톡 | URL 등록 | 영상 페이지 HTML의 `stats` JSON에서 **조회수·좋아요·댓글·공유·저장수·게시일** 전부 수집(키 불필요) + oEmbed 썸네일 + 공식 임베드 재생 |
| 인스타그램 | URL 등록 | `facebookexternalhit` 봇 UA로 공개 게시물 og:description 파싱 → **좋아요·댓글·작성자·게시일·캡션** 수집(조회수·공유는 미공개) + 공식 임베드 |
| 스레드 | URL 등록 | 봇 UA로 og 파싱 → 작성자·본문·썸네일(수치 지표는 공식 API 필요, 0 유지) + 공식 임베드 |

> URL 등록으로 얻는 실지표 (실측 검증 완료)

| 플랫폼 | 조회수 | 좋아요 | 댓글 | 공유 | 저장 | 게시일 | 작성자·캡션 | 임베드재생 |
|--------|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|
| 틱톡 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 인스타그램 | ❌ | ✅ | ✅ | ❌ | ❌ | ✅ | ✅ | ✅ |
| 스레드 | ❌ | ❌ | ❌ | ❌ | ❌ | △ | ✅ | ✅ |

인스타그램은 조회수를 공개하지 않으므로 좋아요를 대표 지표로 표시하고 인게이지먼트율은 `–`로 둡니다(조회수 없이 계산 불가). 이 방식들은 공개 페이지 파싱이라 각 플랫폼이 마크업을 바꾸면 깨질 수 있으며(유튜브 웹 수집과 동일한 성격), **개별 URL 등록만** 가능하고 트렌딩 목록 자동 크롤링은 여전히 불가합니다.

## 데이터는 기본 100% 실데이터 (데모는 opt-in)

- **기본값**: 데모(샘플) 데이터를 생성하지 않습니다. 대시보드에는 실제로 수집된 영상만 표시됩니다(유튜브 자동 + 등록한 URL). 가짜가 실제 트렌드처럼 표시되지 않습니다.
- 각 플랫폼이 비어 있으면 카드/테이블에 **정직한 빈 상태 안내**(URL 등록 유도)가 뜹니다.
- 특정 영상을 지목하는 인사이트("지금 급상승 중…", "인게이지먼트 1위 영상…")는 **실데이터만** 사용합니다.
- `.env`에서 `DEMO_DATA=1`로 켜면 틱톡/인스타/스레드에 파이프라인 시연용 샘플이 생성되며, 각 카드에 **"데모" 배지**가 붙고 인사이트에 데모 포함 경고가 표시됩니다.

**트렌딩 목록 자동 크롤링 불가 (실측 확인):** 틱톡 해시태그/검색/Explore 페이지와 Creative Center API는 모두 서명된 XHR·안티봇 토큰을 요구해 서버 사이드 수집이 불가하고, 인스타/스레드는 로그아웃 상태에서 게시물 목록을 노출하지 않습니다. 따라서 이 세 플랫폼은 **개별 URL 등록**이 유일한 키리스 실데이터 경로입니다. 대량·안정 수집이 필요하면 공식 API(TikTok Research / Instagram Graph / Threads) 승인이 정공법입니다.

## 프로젝트 구조

```
trend-radar/
├── server.js               # Express 서버 + API + 자동 수집 스케줄러
├── src/
│   ├── collectors/
│   │   ├── index.js        # 수집 오케스트레이터 (실데이터 우선, 데모는 DEMO_DATA=1일 때만)
│   │   ├── youtube.js      # 유튜브 공식 API 수집기 (키 설정 시)
│   │   ├── youtube-web.js  # 유튜브 공개 검색 파서 (키 불필요, 조회수·좋아요·댓글·게시일)
│   │   ├── register.js     # URL 등록 수집기 (틱톡 stats·인스타 og·스레드 og 실지표)
│   │   └── demo.js         # 데모(샘플) 엔진 — 기본 비활성, opt-in 시연용
│   ├── classify.js         # 12개 분야 자동 분류기
│   ├── analyze.js          # 트렌드 분석 (인게이지먼트, 급상승, 키워드, 인사이트)
│   └── store.js            # JSON 저장소 (영상 + 시계열 스냅샷)
├── public/                 # 대시보드 (HTML/CSS/JS + Chart.js)
└── data/db.json            # 수집 데이터 (자동 생성)
```

## API

- `GET /api/dashboard?platform=&category=` — KPI, 분야/플랫폼 통계, 키워드, 성장 곡선, 인사이트
- `GET /api/videos?platform=&category=&sort=&limit=` — 영상 목록
- `POST /api/collect` — 즉시 수집 실행
- `POST /api/register` `{urls: [...]}` — 영상 URL 등록 (키 불필요)
- `GET /api/thumb?url=` — 썸네일 프록시 (유튜브·틱톡 CDN 한정)
